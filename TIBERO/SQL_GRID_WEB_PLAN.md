## 1. 개요
- SQL(약 100줄 규모)을 텍스트 영역에 붙여넣고 **[실행]** 버튼을 누르면
- 대상 DB(**PostgreSQL 9.6**)에서 쿼리를 수행하고
- 결과를 **그리드(표) 형식**으로 화면에 조회하는 웹 페이지

## 2. 기술 스택
- **Frontend/Backend**: Python **Streamlit** (기존 `MIG_LIST/app.py` 와 동일 스택)
- **DB 연결**: PostgreSQL 9.6
  - 드라이버 (택 1)
    - [ ] `psycopg2` (`psycopg2-binary`) — PostgreSQL 표준, 9.6 완전 호환 (권장)
    - [ ] `SQLAlchemy` + `psycopg2` (엔진/커넥션 풀 관리)
    - [ ] 기타: ____________________
  - 접속 정보 관리: `.env` 또는 `PG.env` 파일에서 로딩 (host/port/dbname/user/password)
  - ※ PostgreSQL 9.6 은 구버전이므로, 드라이버/라이브러리는 9.6 을 지원하는 버전으로 고정 권장
- **그리드 표시** (택 1)
  - [ ] 기본 `st.dataframe` (정렬/스크롤/CSV 복사 기본 제공, 가장 단순)
  - [ ] `streamlit-aggrid` (AG Grid: 컬럼 필터/페이징/엑셀 내보내기 등 고급 기능)

## 3. 기능 요구사항
1. SQL 입력
   - 여러 줄(100줄+) 입력 가능한 텍스트 영역
   - [실행] 버튼
2. SQL 실행
   - 붙여넣은 SQL 을 대상 DB 에 전송하여 실행
   - 다중 문장 처리 방식 (택 1)
     - [ ] 단일 `SELECT` 문만 허용
     - [ ] `;` 로 구분된 여러 문장 허용 (마지막/각 결과 표시)
       - 참고: psycopg2 는 한 번의 execute 로 여러 문장 실행 시 마지막 결과셋만 반환
3. 결과 표시
   - 결과를 그리드로 표시 (컬럼 헤더 포함)
   - 조회 건수, 수행 시간 표시
   - [ ] 결과 CSV/Excel 다운로드 버튼 제공
4. 오류 처리
   - SQL 오류 시 에러 메시지를 화면에 표시 (앱은 중단되지 않음)

## 4. 보안 / 제약 (★ 반드시 결정)
> 임의 SQL 실행은 "누구나 DB에 무엇이든 실행" 을 의미하므로 범위를 반드시 정해야 합니다.

- 허용할 SQL 범위 (택 1)
  - [ ] 조회 전용 — `SELECT` (및 `WITH ... SELECT`) 만 허용, 그 외 차단
  - [ ] DML 까지 허용 — `INSERT/UPDATE/DELETE` 포함
  - [ ] 제한 없음 — `DDL/DROP/TRUNCATE` 등 모두 허용 (위험)
- 실행 DB 계정
  - [ ] 읽기 전용(read-only) 계정 사용 (권장)
  - [ ] 일반 계정 사용
- 결과 행 수 제한 (PostgreSQL 은 `LIMIT n` 사용)
  - [ ] 자동 제한 적용 — 최대 __________ 행 (예: 1000)
  - [ ] 제한 없음
- 접근 통제
  - [ ] 사내망 전용 (인증 없음)
  - [ ] 로그인/비밀번호 필요
  - [ ] 기타: ____________________
- 실행 이력 로깅
  - [ ] 실행한 SQL / 사용자 / 시각 로그 남김
  - [ ] 남기지 않음
